Los **elementos HTML** son los bloques de construcción fundamentales de cualquier página web. Cada elemento está compuesto por etiquetas que definen su propósito y comportamiento.

<Aside type="tip">
🏷️ **Concepto clave:** Un elemento HTML = Etiqueta de apertura + Contenido + Etiqueta de cierre. Ejemplo: `<p>Contenido</p>`
</Aside>

---

## 🏷️ Etiquetas y su Sintaxis

### Anatomía de una etiqueta HTML

Una etiqueta HTML tiene una estructura específica que debes conocer:

```html
<nombre atributo="valor">Contenido</nombre>
  ↑        ↑        ↑        ↑        ↑
  |        |        |        |        └─ Etiqueta de cierre
  |        |        |        └────────── Contenido del elemento
  |        |        └─────────────────── Valor del atributo
  |        └──────────────────────────── Atributo
  └───────────────────────────────────── Etiqueta de apertura
```

### Tipos de etiquetas

<Tabs>
<TabItem label="Etiquetas con cierre">

**Etiquetas que requieren apertura y cierre:**

```html
<!-- Párrafo -->
<p>Este es un párrafo de texto.</p>

<!-- Título -->
<h1>Este es un título</h1>

<!-- Enlace -->
<a href="https://ejemplo.com">Visitar sitio</a>

<!-- División -->
<div>Contenedor genérico</div>

<!-- Span -->
<span>Texto en línea</span>

<!-- Lista -->
<ul>
  <li>Elemento 1</li>
  <li>Elemento 2</li>
</ul>

<!-- Tabla -->
<table>
  <tr>
    <td>Celda</td>
  </tr>
</table>
```

**Características:**
- ✅ Tienen etiqueta de apertura `<tag>`
- ✅ Tienen etiqueta de cierre `</tag>`
- ✅ Pueden contener texto u otros elementos
- ✅ Son la mayoría de las etiquetas HTML

</TabItem>

<TabItem label="Etiquetas auto-cerradas">

**Etiquetas que no requieren cierre (self-closing):**

```html
<!-- Imagen -->
<img src="imagen.jpg" alt="Descripción">

<!-- Salto de línea -->
<br>

<!-- Línea horizontal -->
<hr>

<!-- Input de formulario -->
<input type="text" placeholder="Nombre">

<!-- Meta información -->
<meta charset="UTF-8">

<!-- Enlace a hoja de estilos -->
<link rel="stylesheet" href="estilos.css">

<!-- Área de mapa -->
<area shape="rect" coords="0,0,100,100" href="link.html">

<!-- Columna de tabla -->
<col span="2">

<!-- Incrustar contenido -->
<embed src="archivo.pdf">

<!-- Parámetro -->
<param name="autoplay" value="true">

<!-- Fuente multimedia -->
<source src="video.mp4" type="video/mp4">

<!-- Track de subtítulos -->
<track src="subtitulos.vtt" kind="subtitles">

<!-- Salto de palabra -->
<wbr>
```

**Características:**
- ✅ No tienen etiqueta de cierre
- ✅ No pueden contener contenido
- ✅ Todos sus datos van en atributos
- ✅ También llamadas "void elements"

<Aside type="note">
💡 **Nota:** En HTML5, puedes escribir `<br>` o `<br />`. Ambas formas son válidas, aunque `<br>` es más común.
</Aside>

</TabItem>

|----------|-------------|---------|

| `hidden` | Ocultar elemento | `hidden` |
| `contenteditable` | Editable | `contenteditable="true"` |
| `draggable` | Arrastrable | `draggable="true"` |
| `lang` | Idioma | `lang="es"` |
| `tabindex` | Orden de tabulación | `tabindex="1"` |

## 🎯 Etiquetas Semánticas vs No Semánticas

### ¿Qué es la semántica en HTML?

La **semántica** se refiere al significado de las etiquetas. Una etiqueta semántica describe claramente su propósito tanto para el navegador como para el desarrollador.

<Tabs>
<TabItem label="Comparación">

### Etiquetas No Semánticas ❌

**No describen su contenido:**

```html
<!-- Div genérico - no dice nada sobre su contenido -->
<div>
  <div>
    <div>Título del sitio</div>
    <div>
      <div>Inicio</div>
      <div>Servicios</div>
    </div>
  </div>
  
  <div>
    <div>Artículo principal</div>
  </div>
  
  <div>Copyright 2024</div>
</div>

<!-- Span genérico - no indica propósito -->
<span>Texto importante</span>
<span>Fecha: 15/01/2024</span>
```

**Problemas:**
- ❌ No se entiende el propósito del contenido
- ❌ Difícil de mantener
- ❌ Malo para SEO
- ❌ Mala accesibilidad

### Etiquetas Semánticas ✅

**Describen claramente su contenido:**

```html
<!-- Estructura semántica clara -->
<body>
  <header>
    <h1>Título del sitio</h1>
    <nav>
      <a href="#inicio">Inicio</a>
      <a href="#servicios">Servicios</a>
    </nav>
  </header>
  
  <main>
    <article>
      <h2>Artículo principal</h2>
      <p>Contenido...</p>
    </article>
  </main>
  
  <footer>
    <p>Copyright 2024</p>
  </footer>
</body>

<!-- Etiquetas semánticas específicas -->
<strong>Texto importante</strong>
<time datetime="2024-01-15">15/01/2024</time>
```

**Beneficios:**
- ✅ Código autodescriptivo
- ✅ Fácil de mantener
- ✅ Mejor SEO
- ✅ Mayor accesibilidad

</TabItem>

## 💬 Comentarios en HTML

Los **comentarios** son notas en el código que el navegador ignora. Son útiles para documentar y organizar el código.

### Sintaxis de comentarios

```html
<!-- Este es un comentario de una línea -->

<!-- 
  Este es un comentario
  de múltiples líneas
  que puede abarcar varias líneas
-->

<!-- Comentario con información útil -->
<header>
  <!-- Navegación principal del sitio -->
  <nav>
    <a href="#inicio">Inicio</a>
  </nav>
</header>
```

<Tabs>
<TabItem label="Usos comunes">

### Usos prácticos de los comentarios

**1. Documentar secciones** 📝

```html
<!-- ==================== -->
<!-- HEADER - Encabezado  -->
<!-- ==================== -->
<header>
  <h1>Mi Sitio</h1>
</header>

<!-- ==================== -->
<!-- MAIN - Contenido     -->
<!-- ==================== -->
<main>
  <p>Contenido principal</p>
</main>

<!-- ==================== -->
<!-- FOOTER - Pie         -->
<!-- ==================== -->
<footer>
  <p>Copyright 2024</p>
</footer>
```

**2. Desactivar código temporalmente** 🔧

```html
<!-- Código desactivado temporalmente
<div class="banner-promocional">
  <p>¡Oferta especial!</p>
</div>
-->

<div class="contenido-principal">
  <p>Contenido activo</p>
</div>
```

**3. Dejar notas para el equipo** 👥

```html
<!-- TODO: Agregar imagen del producto -->
<div class="producto">
  <h3>Nombre del producto</h3>
  <!-- FIXME: El precio debe venir de la base de datos -->
  <p>Precio: $99.99</p>
</div>

<!-- NOTE: Esta sección se actualiza cada semana -->
<section class="noticias">
  <h2>Últimas noticias</h2>
</section>
```

**4. Marcar inicio y fin de secciones** 🏁

```html
<!-- START: Sección de servicios -->
<section id="servicios">
  <h2>Nuestros Servicios</h2>
  <div class="servicio">...</div>
  <div class="servicio">...</div>
</section>
<!-- END: Sección de servicios -->

<!-- START: Sección de contacto -->
<section id="contacto">
  <h2>Contacto</h2>
  <form>...</form>
</section>
<!-- END: Sección de contacto -->
```

**5. Explicar código complejo** 🧩

```html
<!-- 
  Este formulario usa validación HTML5
  - required: campo obligatorio
  - pattern: validación con expresión regular
  - minlength: longitud mínima
-->
<form>
  <input 
    type="email" 
    required 
    pattern="[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}$"
  >
</form>
```

## 🎯 Anidamiento Correcto de Etiquetas

El **anidamiento** se refiere a colocar elementos dentro de otros elementos. Debe hacerse correctamente para que el HTML sea válido.

### Reglas de anidamiento

<Tabs>
<TabItem label="Reglas básicas">

### ✅ Anidamiento correcto

```html
<!-- ✅ CORRECTO: Cierre en orden inverso -->
<div>
  <p>
    <strong>Texto en negrita</strong>
  </p>
</div>

<!-- ✅ CORRECTO: Elementos en línea dentro de bloque -->
<p>
  Este es un <strong>texto importante</strong> con <em>énfasis</em>.
</p>

<!-- ✅ CORRECTO: Listas anidadas -->
<ul>
  <li>Elemento 1</li>
  <li>Elemento 2
    <ul>
      <li>Sub-elemento 2.1</li>
      <li>Sub-elemento 2.2</li>
    </ul>
  </li>
  <li>Elemento 3</li>
</ul>

<!-- ✅ CORRECTO: Estructura semántica -->
<article>
  <header>
    <h2>Título del artículo</h2>
    <p>Fecha: <time datetime="2024-01-15">15/01/2024</time></p>
  </header>
  <p>Contenido del artículo...</p>
  <footer>
    <p>Autor: Juan Pérez</p>
  </footer>
</article>
```

### ❌ Anidamiento incorrecto

```html
<!-- ❌ INCORRECTO: Cierre en orden equivocado -->
<div>
  <p>
    <strong>Texto</p>
  </strong>
</div>

<!-- ❌ INCORRECTO: Elementos de bloque dentro de inline -->
<span>
  <div>Esto no es válido</div>
</span>

<!-- ❌ INCORRECTO: <p> no puede contener otros <p> -->
<p>
  Párrafo 1
  <p>Párrafo 2</p>
</p>

<!-- ❌ INCORRECTO: <a> no puede contener otro <a> -->
<a href="#">
  Enlace externo
  <a href="#">Enlace interno</a>
</a>

<!-- ❌ INCORRECTO: Etiquetas cruzadas -->
<strong><em>Texto</strong></em>
```

</TabItem>

<TabItem label="Elementos de bloque vs inline">

### Elementos de Bloque (Block)

**Ocupan todo el ancho disponible y comienzan en nueva línea:**

```html
<!-- Elementos de bloque -->
<div>División</div>
<p>Párrafo</p>
<h1>Título</h1>
<ul>Lista</ul>
<section>Sección</section>
<article>Artículo</article>
<header>Encabezado</header>
<footer>Pie</footer>
<nav>Navegación</nav>
<main>Principal</main>
<aside>Lateral</aside>
<form>Formulario</form>
<table>Tabla</table>
<blockquote>Cita</blockquote>
<pre>Preformateado</pre>
<hr>
```

**Reglas:**
- ✅ Pueden contener elementos inline
- ✅ Pueden contener otros elementos de bloque
- ✅ Comienzan en nueva línea

### Elementos en Línea (Inline)

**Ocupan solo el espacio necesario, no rompen la línea:**

```html
<!-- Elementos inline -->
<span>Span</span>
<a href="#">Enlace</a>
<strong>Negrita</strong>
<em>Cursiva</em>
<code>Código</code>
<img src="img.jpg" alt="Imagen">
<input type="text">
<button>Botón</button>
<label>Etiqueta</label>
```

- ❌ NO pueden contener elementos de bloque
- ✅ No rompen la línea

### Cómo validar tu HTML

**Herramientas de validación:**

1. **W3C Validator** 🔍
   - URL: https://validator.w3.org/
   - Valida sintaxis y anidamiento
   - Detecta errores comunes

2. **Extensiones de navegador** 🔧
   - HTML Validator (Chrome/Firefox)
   - Valida en tiempo real

3. **DevTools del navegador** 🛠️
   - F12 → Console
   - Muestra advertencias de HTML

**Errores comunes detectados:**

```html

<!-- Error: Etiqueta desconocida -->
```

## 🎓 Resumen

<Aside type="note">
**Puntos clave de esta lección:**

### Etiquetas y Sintaxis
✅ **Etiquetas con cierre**: `<p>Contenido</p>`  
✅ **Etiquetas auto-cerradas**: `<img>`, `<br>`, `<hr>`  
✅ **Atributos**: `<a href="url" target="_blank">`  
✅ **Atributos globales**: `id`, `class`, `style`, `title`  

### Semántica
✅ **Etiquetas semánticas** describen su contenido ⭐  
✅ **Mejor SEO** y accesibilidad  
✅ Usar `<header>`, `<nav>`, `<main>`, `<article>`, `<footer>`  
✅ Evitar `<div>` y `<span>` genéricos cuando hay alternativa  

### Comentarios
✅ **Sintaxis**: `<!-- Comentario -->`  
✅ Documentar secciones y código  
✅ Desactivar código temporalmente  
✅ No abusar de comentarios obvios  

### Anidamiento
✅ **Cerrar en orden inverso** ⭐  
✅ Bloque puede contener bloque + inline  
✅ Inline solo puede contener inline  
✅ Validar con W3C Validator  

### Patrones comunes
```html
<!-- Estructura básica -->
<article>
  <header>
    <h2>Título</h2>
  </header>
  <p>Contenido...</p>
  <footer>Autor</footer>
</article>

<!-- Texto con formato -->
<p>
  Este es <strong>importante</strong> y 
  <em>con énfasis</em>.
</p>

<!-- Lista anidada -->
<ul>
  <li>Item 1</li>
  <li>Item 2
    <ul>
      <li>Sub-item</li>
    </ul>
  </li>
</ul>
```
