<Tabs>

### Concepto de m√≥dulo

Un **m√≥dulo** es un archivo de Python (`.py`) que contiene definiciones de funciones, clases y variables que pueden ser reutilizadas en otros programas. Es la unidad b√°sica de organizaci√≥n de c√≥digo en Python.

### Concepto de librer√≠a (biblioteca)

Una **librer√≠a** (o biblioteca) es una colecci√≥n de m√≥dulos relacionados que proporcionan funcionalidad espec√≠fica. Puede ser un solo m√≥dulo o un conjunto de m√≥dulos organizados en paquetes.

### Diferencias clave

**M√≥dulo**: Un archivo `.py` individual  
**Paquete**: Directorio que contiene m√∫ltiples m√≥dulos y un archivo `__init__.py`  
**Librer√≠a**: T√©rmino general para m√≥dulos y paquetes que proporcionan funcionalidad  

### Prop√≥sito

**üîπ Reutilizaci√≥n**: Evitar reescribir c√≥digo com√∫n  
**üîπ Organizaci√≥n**: Estructurar proyectos grandes en partes manejables  
**üîπ Mantenibilidad**: Facilitar actualizaciones y correcciones  
**üîπ Colaboraci√≥n**: Compartir c√≥digo entre desarrolladores  
**üîπ Abstracci√≥n**: Ocultar complejidad detr√°s de interfaces simples  

### Tipos de librer√≠as

**Est√°ndar**: Incluidas con Python (math, random, datetime)  
**Externas**: Instaladas con pip (requests, pandas, numpy)  
**Propias**: Creadas por el desarrollador para su proyecto

<Tabs>
<TabItem label="üëî Parte Te√≥rica">

### Sintaxis de import

**import modulo**: Importa el m√≥dulo completo  
**from modulo import funcion**: Importa elementos espec√≠ficos  
**import modulo as alias**: Importa con un alias  
**from modulo import ***: Importa todo (no recomendado)  

### Formas de importar

```python
# Importar m√≥dulo completo
import math

# Importar elementos espec√≠ficos
from math import sqrt, pi

# Importar con alias
import math as m

# Importar todo (evitar)
from math import *
```

### Ventajas de cada forma

**import modulo**: Evita conflictos de nombres, claro de d√≥nde viene cada funci√≥n  
**from modulo import**: C√≥digo m√°s limpio, acceso directo a funciones  
**import as**: Nombres cortos para m√≥dulos con nombres largos  

### Buenas pr√°cticas

**üîπ Imports al inicio**: Colocar todos los imports al principio del archivo  
**üîπ Orden**: Est√°ndar ‚Üí Externos ‚Üí Propios  
**üîπ Evitar ***: No usar `from modulo import *` (contamina namespace)  
**üîπ Alias claros**: Usar alias reconocibles (np para numpy, pd para pandas)  

</TabItem>

### Librer√≠a math

Proporciona funciones matem√°ticas avanzadas: ra√≠ces, potencias, trigonometr√≠a, logaritmos, constantes matem√°ticas.

**Funciones comunes**: `sqrt()`, `pow()`, `ceil()`, `floor()`, `sin()`, `cos()`, `log()`

### Librer√≠a random

Genera n√∫meros aleatorios y realiza selecciones aleatorias.

**Funciones comunes**: `random()`, `randint()`, `choice()`, `shuffle()`, `sample()`  
**Uso**: Juegos, simulaciones, muestreo, testing  

### Librer√≠a datetime

Maneja fechas, horas y operaciones temporales.

**Clases principales**: `date`, `time`, `datetime`, `timedelta`  
**Uso**: Registros, c√°lculos de tiempo, formateo de fechas

### ¬øQu√© es pip?

**pip** (Package Installer for Python) es el gestor de paquetes oficial de Python. Permite instalar, actualizar y desinstalar librer√≠as externas desde PyPI (Python Package Index).

### Comandos principales

**Instalar**: `pip install nombre_libreria`  
**Desinstalar**: `pip uninstall nombre_libreria`  
**Actualizar**: `pip install --upgrade nombre_libreria`  
**Listar instaladas**: `pip list`  
**Ver informaci√≥n**: `pip show nombre_libreria`  

### Librer√≠as externas populares

**requests**: Realizar peticiones HTTP  
**pandas**: An√°lisis de datos  
**numpy**: C√°lculos num√©ricos  
**matplotlib**: Gr√°ficos y visualizaci√≥n  
**flask**: Framework web  
**django**: Framework web completo  

### Entornos virtuales

**Prop√≥sito**: Aislar dependencias por proyecto  
**Comando**: `python -m venv nombre_entorno`  
**Activar**: `nombre_entorno\Scripts\activate` (Windows)  

</TabItem>
