### Reglas obligatorias (sintaxis)

Python impone reglas estrictas que deben cumplirse para que un nombre de variable sea vÃ¡lido:

**âœ… Regla 1: Caracteres permitidos**  
Los nombres solo pueden contener letras (a-z, A-Z), nÃºmeros (0-9) y guiones bajos (_).

**âœ… Regla 2: Primer carÃ¡cter**  
Debe comenzar con una letra o guiÃ³n bajo, nunca con un nÃºmero.

**âœ… Regla 3: Sensibilidad a mayÃºsculas**  
Python distingue entre mayÃºsculas y minÃºsculas. `edad`, `Edad` y `EDAD` son tres variables diferentes.

**âœ… Regla 4: Palabras reservadas**  
No se pueden usar palabras clave de Python como nombres de variables (`if`, `for`, `while`, `class`, `def`, etc.).

**âœ… Regla 5: Sin espacios**  
Los nombres no pueden contener espacios. Usa guiones bajos para separar palabras.

### Convenciones recomendadas (estilo PEP 8)

AdemÃ¡s de las reglas obligatorias, existen convenciones de estilo que mejoran la legibilidad:

**ğŸ”¹ snake_case para variables**  
Usa minÃºsculas con guiones bajos para separar palabras: `nombre_completo`, `edad_usuario`, `precio_total`.

**ğŸ”¹ Nombres descriptivos**  
Elige nombres que expliquen claramente quÃ© contiene la variable: `temperatura_maxima` en lugar de `temp` o `t`.

**ğŸ”¹ Evitar abreviaturas confusas**  
Prefiere `numero_estudiantes` sobre `num_est`, a menos que la abreviatura sea universalmente conocida.

**ğŸ”¹ Constantes en MAYÃšSCULAS**  
Para valores que no cambian, usa mayÃºsculas: `PI = 3.14159`, `MAX_INTENTOS = 3`.

**ğŸ”¹ Evitar nombres de una letra**  
Excepto en bucles (`i`, `j`, `k`) o coordenadas (`x`, `y`, `z`), usa nombres mÃ¡s descriptivos.
