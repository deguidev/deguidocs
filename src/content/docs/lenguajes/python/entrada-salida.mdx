<Tabs>

### Concepto de input()

La funci√≥n **`input()`** es una funci√≥n integrada de Python que permite capturar datos ingresados por el usuario a trav√©s del teclado. Esta funci√≥n pausa la ejecuci√≥n del programa, espera a que el usuario escriba algo y presione Enter, luego retorna el texto ingresado como una cadena de texto (string).

### Sintaxis

```python
variable = input(mensaje_opcional)
```

**Par√°metros:**
- **mensaje_opcional**: Texto que se muestra al usuario como indicaci√≥n de qu√© debe ingresar (prompt).

**Retorno:**
- Siempre retorna un **string** (cadena de texto), sin importar qu√© escriba el usuario.

### Caracter√≠sticas importantes

**üîπ Retorna siempre string**  
Incluso si el usuario escribe n√∫meros, `input()` los retorna como texto. Es necesario convertirlos expl√≠citamente usando `int()`, `float()`, etc.

**üîπ Pausa la ejecuci√≥n**  
El programa se detiene hasta que el usuario ingrese datos y presione Enter.

**üîπ Mensaje opcional**  
Aunque el mensaje es opcional, es buena pr√°ctica incluirlo para indicar al usuario qu√© debe ingresar.

**üîπ Captura toda la l√≠nea**  
`input()` captura todo lo que el usuario escriba hasta presionar Enter, incluyendo espacios.

### Prop√≥sito y aplicaciones

- **Interactividad**: Permite crear programas que interact√∫an con el usuario
- **Recopilaci√≥n de datos**: Capturar informaci√≥n necesaria para procesar
- **Personalizaci√≥n**: Adaptar el comportamiento del programa seg√∫n la entrada del usuario
- **Validaci√≥n**: Solicitar datos hasta que sean correctos

## üñ®Ô∏è 4.2 Mostrar resultados con print() y f-strings

<Tabs>
<TabItem label="üëî Parte Te√≥rica">

### La funci√≥n print()

La funci√≥n **`print()`** muestra informaci√≥n en la consola. Es la forma principal de comunicar resultados al usuario en programas de consola.

### F-strings (Formatted String Literals)

Las **f-strings** son una forma moderna y eficiente de formatear cadenas en Python (disponibles desde Python 3.6+). Se crean anteponiendo una `f` o `F` antes de las comillas y permiten insertar expresiones dentro de llaves `{}`.

**Sintaxis:**
```python
f"Texto {variable} m√°s texto"
```

### Ventajas de f-strings

**üîπ Legibilidad**  
El c√≥digo es m√°s claro y f√°cil de leer que otros m√©todos de formateo.

**üîπ Eficiencia**  
Son m√°s r√°pidas que otros m√©todos como `.format()` o concatenaci√≥n con `+`.

**üîπ Expresiones dentro**  
Puedes incluir operaciones, llamadas a funciones y expresiones directamente.

**üîπ Sintaxis concisa**  
Menos c√≥digo para lograr el mismo resultado.

### Caracter√≠sticas de print()

- **M√∫ltiples argumentos**: Acepta varios valores separados por comas
- **Separador personalizable**: Par√°metro `sep` para cambiar el separador (por defecto es espacio)
- **Final personalizable**: Par√°metro `end` para cambiar el final (por defecto es salto de l√≠nea)
- **Conversi√≥n autom√°tica**: Convierte autom√°ticamente valores a string

</TabItem>

## üìù 4.3 Formateo de texto con variables (f"", format())

<Tabs>
<TabItem label="üëî Parte Te√≥rica">

### M√©todos de formateo en Python

Python ofrece varios m√©todos para formatear cadenas de texto con variables:

**üîπ F-strings (recomendado)**  
M√©todo moderno, legible y eficiente introducido en Python 3.6.

**üîπ M√©todo .format()**  
M√©todo tradicional que usa llaves `{}` como marcadores de posici√≥n y el m√©todo `.format()` para insertar valores.

**üîπ Operador % (legacy)**  
M√©todo antiguo heredado de C, menos usado en c√≥digo moderno.

### El m√©todo .format()

**Sintaxis:**
```python
"Texto {} m√°s texto {}".format(valor1, valor2)
```

**Caracter√≠sticas:**
- Usa llaves `{}` como marcadores de posici√≥n
- Permite especificar posiciones: `{0}`, `{1}`
- Soporta nombres: `{nombre}`, `{edad}`
- Ofrece opciones de formateo similares a f-strings

### Cu√°ndo usar cada m√©todo

**F-strings**: Para la mayor√≠a de casos en c√≥digo moderno (Python 3.6+)  
**.format()**: Cuando necesitas reutilizar plantillas o trabajas con Python anterior a 3.6  
**% operator**: Solo en c√≥digo legacy o compatibilidad con versiones muy antiguas

</TabItem>

<TabItem label="üí° Ejemplos Pr√°cticos">

### Concepto de validaci√≥n de entradas

La **validaci√≥n de entradas** es el proceso de verificar que los datos ingresados por el usuario cumplan con ciertos criterios antes de procesarlos. Es fundamental para crear programas robustos y evitar errores.

### Importancia de la validaci√≥n

**üîπ Prevenir errores**  
Evita que el programa falle por datos incorrectos o inesperados.

**üîπ Mejorar experiencia del usuario**  
Proporciona retroalimentaci√≥n clara sobre qu√© datos son v√°lidos.

**üîπ Seguridad**  
Protege contra entradas maliciosas o no deseadas.

**üîπ Integridad de datos**  
Asegura que solo se procesen datos correctos y consistentes.

### Tipos de validaci√≥n com√∫n

**Validaci√≥n de tipo**: Verificar que el dato sea del tipo esperado (n√∫mero, texto, etc.)  
**Validaci√≥n de rango**: Verificar que un n√∫mero est√© dentro de l√≠mites aceptables  
**Validaci√≥n de formato**: Verificar que el texto siga un patr√≥n espec√≠fico  
**Validaci√≥n de existencia**: Verificar que el campo no est√© vac√≠o  

### T√©cnicas de validaci√≥n

**üîπ Try-except**  
Capturar errores de conversi√≥n de tipos.

**üîπ Bucles while**  
Repetir la solicitud hasta obtener datos v√°lidos.

**üîπ M√©todos de string**  
Usar `.isdigit()`, `.isalpha()`, `.strip()`, etc.

**üîπ Expresiones condicionales**  
Verificar condiciones espec√≠ficas con if/elif/else.

</TabItem>
