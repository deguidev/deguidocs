## Fundamentos (1-15)

### 1. ¿Qué es JSX?

> Extensión de sintaxis que combina HTML con JavaScript. Se transpila a React.createElement(). No es HTML, es JavaScript.

<Code code={`// JSX
const element = <h1 className="titulo">Hola</h1>;

// Se transpila a:
const element = React.createElement('h1', { className: 'titulo' }, 'Hola');`} lang="jsx" />

### 2. ¿Por qué usamos className en vez de class?

> `class` es palabra reservada en JS. JSX usa `className` para atributo CSS y `htmlFor` en vez de `for`.

### 3. ¿Qué son los Fragments?

> Permiten agrupar elementos sin agregar nodo extra al DOM. Sintaxis: `<React.Fragment>` o `<>`.

<Code code={`// Problema: React necesita un elemento padre
// Solución: Fragment (no agrega nodo al DOM)

// Sintaxis completa
<React.Fragment>
  <h1>Título</h1>
  <p>Párrafo</p>
</React.Fragment>

// Sintaxis corta
<>
  <h1>Título</h1>
  <p>Párrafo</p>
</>`} lang="jsx" />

### 4. ¿Qué es el renderizado condicional?

> Mostrar componentes según condición. Usar: `&&` (si true), ternario `? :`, o return temprano.

<Code code={`function Componente({ isLoggedIn, items }) {
  // Con &&
  return <div>{isLoggedIn && <UserPanel />}</div>;
  
  // Con ternario
  return <div>{isLoggedIn ? <UserPanel /> : <Login />}</div>;
  
  // Con if temprano
  if (!isLoggedIn) return <Login />;
  return <UserPanel />;
  
  // Con variable
  let content;
  if (isLoggedIn) {
    content = <UserPanel />;
  } else {
    content = <Login />;
  }
  return <div>{content}</div>;
}`} lang="jsx" />

### 5. ¿Cómo renderizar listas?

> Usar `.map()` para transformar array en elementos JSX. Cada elemento necesita `key` única.

<Code code={`function Lista({ items }) {

          {item.nombre}
        </li>

}`} lang="jsx" />

### 6. ¿Qué es prop drilling y cómo evitarlo?

> Pasar props a través de muchos niveles. Soluciones: Context API, estado global (Redux/Zustand), composición.

<Code code={`// Prop drilling - pasar props a través de muchos niveles
<Abuelo user={user}>
  <Padre user={user}>
    <Hijo user={user}>
      <Nieto user={user} /> {/* Finalmente lo usa */}
    </Hijo>
  </Padre>
</Abuelo>

// Soluciones:
// 1. Context API
// 2. Estado global (Redux, Zustand)
// 3. Composición de componentes`} lang="jsx" />

### 7. ¿Qué es Context API?

> Sistema para compartir datos sin prop drilling. createContext + Provider + useContext. Para datos globales (tema, usuario).

<Code code={`// 1. Crear contexto
const UserContext = createContext(null);

// 2. Proveer valor
function App() {
  const [user, setUser] = useState({ nombre: 'Juan' });
  
  return (
    <UserContext.Provider value={{ user, setUser }}>
      <MiApp />
    </UserContext.Provider>
  );
}

// 3. Consumir en cualquier nivel
function Nieto() {
  const { user } = useContext(UserContext);
  return <p>Hola, {user.nombre}</p>;
}`} lang="jsx" />

### 8. ¿Cuándo usar Context vs Redux?

| Context | Redux |
|---------|-------|
| Estado simple | Estado complejo |
| Pocos consumidores | Muchos consumidores |
| Actualizaciones poco frecuentes | Actualizaciones frecuentes |
| Sin middleware | Necesitas middleware |

### 10. ¿Qué es useCallback?

<Code code={`function Padre() {
  const [count, setCount] = useState(0);
  
  // Sin useCallback - nueva función en cada render
  const handleClick = () => {
    console.log('click');
  };
  
  // Con useCallback - misma referencia si deps no cambian
  const handleClickMemo = useCallback(() => {
    console.log('click', count);
  }, [count]);
  
  return <Hijo onClick={handleClickMemo} />;
}`} lang="jsx" />

### 11. ¿Qué es useMemo?

<Code code={`function Componente({ items }) {

  // Solo recalcula si 'items' cambia

### 12. ¿Cuál es la diferencia entre useMemo y useCallback?

<Code code={`// useMemo - memoriza un VALOR
const valor = useMemo(() => calcularAlgo(a, b), [a, b]);

// useCallback - memoriza una FUNCIÓN
const fn = useCallback(() => hacerAlgo(a, b), [a, b]);

// useCallback es equivalente a:
const fn = useMemo(() => () => hacerAlgo(a, b), [a, b]);`} lang="jsx" />

### 13. ¿Qué es React.memo?

<Code code={`// Componente que solo re-renderiza si props cambian
const MiComponente = React.memo(function MiComponente({ nombre }) {
  console.log('Renderizando...');
  return <p>{nombre}</p>;
});

// Con comparación personalizada
const MiComponente = React.memo(
  function MiComponente({ user }) {
    return <p>{user.nombre}</p>;
  },
  (prevProps, nextProps) => {
    return prevProps.user.id === nextProps.user.id;
  }
);`} lang="jsx" />

### 14. ¿Qué es useReducer?

<Code code={`const initialState = { count: 0 };

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };
    case 'reset':
      return initialState;
    default:
      throw new Error();
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, initialState);
  
  return (
    <>
      Count: {state.count}
      <button onClick={() => dispatch({ type: 'increment' })}>+</button>
      <button onClick={() => dispatch({ type: 'decrement' })}>-</button>
    </>
  );
}`} lang="jsx" />

### 15. ¿Cuándo usar useState vs useReducer?

| useState | useReducer |
|----------|------------|
| Estado simple | Estado complejo |
| Pocas transiciones | Muchas transiciones |
| Lógica simple | Lógica compleja |
| Independiente | Valores relacionados |

### 18. ¿Cómo hacer fetch de datos correctamente?

<Code code={`function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  
  useEffect(() => {
    let cancelled = false;
    
    async function fetchData() {
      try {
        setLoading(true);
        const response = await fetch(url);
        const json = await response.json();
        
        if (!cancelled) {
          setData(json);
        }
      } catch (err) {
        if (!cancelled) {
          setError(err);
        }
      } finally {
        if (!cancelled) {
          setLoading(false);
        }
      }
    }
    
    fetchData();
    
    return () => {
      cancelled = true;
    };
  }, [url]);
  
  return { data, loading, error };
}`} lang="jsx" />

### 21. ¿Qué son los custom hooks?

<Code code={`// Custom hook - reutiliza lógica con estado
function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    const stored = localStorage.getItem(key);
    return stored ? JSON.parse(stored) : initialValue;
  });
  
  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);
  
  return [value, setValue];
}

// Uso
function App() {
  const [theme, setTheme] = useLocalStorage('theme', 'dark');
}`} lang="jsx" />

### 22. ¿Cuáles son las reglas de los Hooks?

1. **Solo en el nivel superior** - No dentro de loops, condiciones o funciones anidadas
2. **Solo en componentes de React** - O en custom hooks
3. **Mismo orden siempre** - React depende del orden de llamada

<Code code={`// MAL
if (condition) {
  useEffect(() => {});
}

// BIEN
useEffect(() => {
  if (condition) {
    // lógica
  }
});`} lang="jsx" />
