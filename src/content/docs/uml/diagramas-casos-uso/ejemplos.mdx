## 4.1. Sistema de Gestión Bancaria

Un sistema bancario completo que incluye operaciones de clientes, cajeros y administradores.

<PlantUml code={`@startuml
left to right direction
skinparam packageStyle rectangle

actor Cliente
actor Cajero

actor "Sistema de Seguridad" as SistemaSeg

rectangle "Sistema Bancario" {
  usecase "Consultar Saldo" as ConsultarSaldo
  usecase "Realizar Retiro" as RealizarRetiro
  usecase "Realizar Depósito" as RealizarDeposito
  usecase "Transferir Fondos" as TransferirFondos
  usecase "Solicitar Préstamo" as SolicitarPrestamo
  usecase "Abrir Cuenta" as AbrirCuenta
  usecase "Cerrar Cuenta" as CerrarCuenta
  usecase "Generar Reporte" as GenerarReporte
  
  usecase "Autenticar Usuario" as AutenticarUsuario
  usecase "Verificar Fondos" as VerificarFondos
  usecase "Registrar Transacción" as RegistrarTransaccion
  usecase "Validar Identidad" as ValidarIdentidad
  usecase "Consultar Historial Crediticio" as ConsultarHistorial
  usecase "Notificar Actividad Sospechosa" as NotificarSospecha
  usecase "Aplicar Comisión" as AplicarComision

  Cliente --> RealizarRetiro
  Cliente --> RealizarDeposito

  Cajero --> RealizarDeposito

  Cajero --> CerrarCuenta

  Admin --> CerrarCuenta

  RealizarRetiro ..> VerificarFondos : <<include>>
  RealizarRetiro ..> RegistrarTransaccion : <<include>>
  
  RealizarDeposito ..> AutenticarUsuario : <<include>>
  RealizarDeposito ..> RegistrarTransaccion : <<include>>

  TransferirFondos ..> VerificarFondos : <<include>>

@enduml`} />

<Aside type="note">
  Este ejemplo muestra un sistema bancario completo con:
  - **Múltiples actores**: Cliente, Cajero, Administrador y sistemas externos
  - **Relaciones de inclusión**: Autenticación y verificación obligatorias
  - **Relaciones de extensión**: Validaciones y comisiones opcionales
  - **Separación de responsabilidades**: Cada actor tiene casos de uso específicos
</Aside>
