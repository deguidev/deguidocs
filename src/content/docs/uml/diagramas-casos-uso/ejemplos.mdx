## Resumen de Buenas Prácticas

<div>
  <ul>
    <li><strong>Claridad sobre complejidad</strong>: Prefiera diagramas claros y legibles sobre diagramas exhaustivos</li>
    <li><strong>Agrupe casos de uso relacionados</strong>: Use paquetes o rectángulos para organizar funcionalidades</li>
    <li><strong>Nombre descriptivos</strong>: Use verbos en infinitivo para casos de uso (ej: "Realizar Compra")</li>
    <li><strong>Evite redundancia</strong>: Use «include» para comportamiento común</li>
    <li><strong>Documente puntos de extensión</strong>: Indique claramente cuándo ocurren las extensiones</li>
    <li><strong>Valide con stakeholders</strong>: Los diagramas deben ser comprensibles para usuarios no técnicos</li>
    <li><strong>Mantenga consistencia</strong>: Use las mismas convenciones en todos los diagramas del proyecto</li>
  </ul>
</div>
