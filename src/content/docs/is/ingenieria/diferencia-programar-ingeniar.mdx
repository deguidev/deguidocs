Imagina que programar es como cocinar un plato para ti, pero ingeniar software es como ser el chef de un restaurante completo. Ambos cocinan, pero uno piensa en todo el negocio.

## ¿Qué es programar?

<Aside type="tip">
**Programar** es como escribir una receta y cocinarla. Te enfocas en hacer que funcione, punto.
</Aside>

Cuando solo programamos:
- Escribimos código para resolver un problema específico
- Trabajamos solos o en grupos pequeños
- Nos enfocamos en que el programa funcione
- No pensamos mucho en el futuro

## ¿Qué es ingeniar software?

<Aside type="note">
**Ingeniar software** es como planear, construir y administrar un restaurante completo. Piensas en todo: los clientes, los empleados, el dinero, el futuro.
</Aside>

Cuando ingeniamos software:
- Planeamos todo el proyecto desde el inicio
- Trabajamos en equipos grandes
- Pensamos en la calidad, costos y tiempo
- Preparamos el software para crecer y cambiar

## Ejemplos del mundo real

### Solo Programar:
- Hacer una calculadora para tu tarea
- Crear un programa para organizar tus fotos
- Automatizar algo repetitivo en tu trabajo

### Ingeniar Software:
- Crear Netflix (millones de usuarios)
- Desarrollar el sistema de un banco
- Hacer una app como Uber (conductores + pasajeros)

## ¿Por qué es importante la diferencia?

<Aside type="caution">
**¡Cuidado!** Si intentas hacer un proyecto grande solo programando (sin planear), es como intentar construir un edificio sin arquitecto. Puede colapsar.
</Aside>

Las consecuencias de no ingeniar cuando es necesario:

1. **El proyecto se vuelve un desastre** - Como una cocina sucia
2. **Cuesta mucho dinero arreglarlo** - Como reconstruir una casa mal hecha
3. **Los usuarios se frustran** - Como clientes que se van del restaurante
4. **El equipo se estresa** - Como cocineros trabajando sin organización

## Resumen Simple

<Badge text="Programar" variant="tip" /> = Hacer que funcione

<Badge text="Ingeniar" variant="success" /> = Hacer que funcione bien, a tiempo, sin gastar mucho, y que dure

**La clave:** Ambos son importantes. Necesitas saber programar para poder ingeniar software, pero si quieres hacer cosas grandes, necesitas aprender a ingeniar.
